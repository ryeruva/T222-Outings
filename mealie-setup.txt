MEALIE SETUP INSTRUCTIONS — Troop 222 Outing Coordinator
==========================================================
Prerequisites: Ubuntu machine with internet access


STEP 1 — INSTALL DOCKER
------------------------
Run these commands in your terminal (copy the whole block):

    sudo apt-get update
    sudo apt-get install -y ca-certificates curl
    sudo install -m 0755 -d /etc/apt/keyrings
    sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
    sudo chmod a+r /etc/apt/keyrings/docker.asc
    echo \
      "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
      https://download.docker.com/linux/ubuntu \
      $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
      sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    sudo apt-get update
    sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

Add yourself to the docker group (so you don't need sudo every time):

    sudo usermod -aG docker $USER
    newgrp docker


STEP 2 — CREATE THE MEALIE COMPOSE FILE
-----------------------------------------
Run this to create the folder and config file:

    mkdir -p ~/mealie && cat > ~/mealie/docker-compose.yml << 'EOF'
    version: "3.7"
    services:
      mealie:
        image: ghcr.io/mealie-recipes/mealie:latest
        container_name: mealie
        restart: unless-stopped
        ports:
          - "9000:9000"
        environment:
          - ALLOW_SIGNUP=false
          - TZ=America/Los_Angeles
          - BASE_URL=http://localhost:9000
        volumes:
          - mealie-data:/app/data

    volumes:
      mealie-data:
    EOF

    NOTE: Change TZ to your timezone if needed (e.g. America/New_York, America/Chicago).


STEP 3 — START MEALIE
----------------------
    cd ~/mealie && docker compose up -d

Wait about 20 seconds, then open http://localhost:9000 in your browser.


STEP 4 — FIRST LOGIN
---------------------
Default credentials (change these immediately after logging in):

    Email:    changeme@example.com
    Password: MyPassword

Go to Profile > Security and update both.


STEP 5 — GET AN API TOKEN
--------------------------
1. In Mealie, click your avatar in the top-right corner
2. Click Profile
3. Scroll down to API Tokens
4. Click Generate
5. Name it "T222 Read-Only"
6. Copy the token — you will not see it again


STEP 6 — WIRE IT INTO YOUR HTML FILES
--------------------------------------
Edit the following two constants near the top of the <script> block
in BOTH of these files:

    /home/ubuntu/spring-prep-hike/Mt_Tallac_Menu.html
    /home/ubuntu/spring-prep-hike/Mt_Tallac_Expenses.html

Change:
    const MEALIE_BASE  = '';
    const MEALIE_TOKEN = '';

To:
    const MEALIE_BASE  = 'http://localhost:9000';
    const MEALIE_TOKEN = 'paste-your-token-here';


STEP 7 — ADD THE MEAL PLAN IN MEALIE
--------------------------------------
1. Go to http://localhost:9000 and log in
2. Click Meal Planner in the left sidebar
3. Navigate to May 2026
4. Add meals for May 8, 9, and 10
5. Search for existing recipes or create new ones as you go
6. To show allergy badges on the menu page, add Tags to each recipe
   (e.g. nut-free, gluten-free, dairy-free, vegan, vegetarian)


STEP 8 — VIEW YOUR PAGES
--------------------------
Because the HTML files are on your local filesystem, you must serve
them through a local web server or the browser will block API calls.

Run this once in a terminal:

    cd /home/ubuntu/spring-prep-hike && python3 -m http.server 8080

Then open these URLs in your browser:

    http://localhost:8080/Mt_Tallac_Menu.html
    http://localhost:8080/Mt_Tallac_Expenses.html


USEFUL DOCKER COMMANDS
-----------------------
Stop Mealie:
    docker compose -f ~/mealie/docker-compose.yml down

Start Mealie again:
    docker compose -f ~/mealie/docker-compose.yml up -d

Check logs if something is wrong:
    docker logs mealie

Check if Mealie is running:
    docker ps


TROUBLESHOOTING
---------------
- If http://localhost:9000 does not load after 30 seconds, run:
      docker logs mealie
  and look for error messages.

- If your menu page shows "Could not load meal plan", confirm that:
  1. Mealie is running (docker ps shows the container)
  2. MEALIE_BASE and MEALIE_TOKEN are set correctly in the HTML file
  3. You are viewing the page via http://localhost:8080, not file://

- If you get a login error, the default password may have already
  been changed. Reset it by running:
      docker exec -it mealie python /app/mealie/scripts/change_password.py
